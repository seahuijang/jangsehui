## 0. 사용자 유형 정의

| 유형 | 정의 | 주요 권한 |
|------|------|----------|
| **컬리 임직원 - MD/AMD** | 공급사 등록·관리 주체 | 공급사 등록·수정 (버튼 노출), 담당MD/AMD 지정 |
| **컬리 임직원 - CO** | 공급사 등록 + 운영 보조 | 공급사 등록·수정 (버튼 노출) |
| **컬리 임직원 - 별도 지정 담당자** | 대량 등록·수정 권한자 | 엑셀 대량 등록/수정(최대 500건), 입고시간 대량수정(최대 500건) |
| **컬리 임직원 - 망분리 PC 담당자** | 정보 다운로드 권한자 | 공급사정보 다운로드 |
| **컬리 임직원 - 권한관리자** | 계정·권한 운영자 | 계정별 권한 설정, 권한그룹 운영 |
| **컬리 임직원 - KLS 운영자** | 화주사관리 담당 | 화주사 등록·공급사 매핑·담당자 설정 |
| **공급사 담당자(외부 운영자)** | 공급사에 등록되는 외부 인물 | 시스템 사용자 아님 — 발주·반품·정산 알림 수신처 |
| **시스템(SYSTEM)** | 자동·외부 연동 트리거 | 알림톡 발송·팝빌 연동·SSO 미완 이중등록 |
| **외부 - 팝빌(POPBILL)** | 전자세금계산서 시스템 | 세금계산서 역/정발행, 국세청 전송 |
| **외부 - 카카오 알림톡** | 메시지 채널 | 발주·반품·정산·프로모션 알림 발송 |
| **외부 - 그룹웨어** | 권한 신청 채널 | 파트너포털 계정/권한 신청서 |

## 2. 컬리 임직원 시나리오 - MD/AMD/CO (공급사관리)

### 2-1. [정상] 공급사 단건 등록

| 항목 | 내용 |
|------|------|
| 진입 경로 | Admin > 공급사관리 > 공급사 등록 |
| 사용자 행동 | 1. 사업자번호 입력<br>2. 기본정보 입력<br>3. 담당MD·담당AMD·발주담당 지정<br>4. 공급사 담당자 최소 1명 등록<br>5. 팝빌 연동 정보 입력<br>6. [등록] |
| 시스템 처리 | 공급사 마스터 생성, 기본값 설정 (AP기간=10일/월3회, 지급일=60일) |
| 정책 근거 | §1 |

| 정책 근거 | §1 |

## 11. 공급사·화주사 등록 ASCII 흐름도

```
[MD/AMD/CO: 공급사 등록]
    │
    ▼ 사업자번호 검증
    ├─ 휴·폐업 → 차단
    ├─ 중복 → 차단 (1:1 원칙)
    └─ 정상
         │
         ▼ 필수 입력
         ├─ 담당MD / 담당AMD / 발주담당 (모두 필수)
         ├─ 공급사 담당자 ≥ 1명
         ├─ 팝빌 연동 정보 (정산 위해 필수)
         └─ 기본값 적용 (AP기간=10일/월3회, 지급일=60일)
              │
              ▼
         [공급사 마스터 생성]
              │
              ├─ [별도 지정자]: 엑셀 대량(≤500건)·입고시간 대량수정(≤500건)
              ├─ [망분리 PC 담당자]: 정보 다운로드
              └─ 판매관리 '사용함' 설정 (팀 내 검토절차 필수)


[KLS 운영자: 화주사 등록]
    │
    ▼ 서비스구분 선택 (FBK / 3PL)  ⚠️ 한 번 저장 후 수정 불가
    │
    ▼ 사업자번호 (인증 실패해도 등록 가능)
    │
    ▼ 파트너 연계코드 (3P파트너 UUID, 컬리몰 입점 시)
    │
    ▼ 담당자 등록 (대표운영자 포함 ≤ 5명)
    │
    ▼ 화주사 > 공급사 매핑 (≥ 1개, 승인절차 없음)
    │
    ▼ 권한 신청: 그룹웨어 > 파트너포털 계정/권한 신청서 (기본권한: KLS)


[시스템 외부 연동]
    ├─ 카카오 알림톡 → 발주생성·반품요청·회수지연·정산승인요청·프로모션 합의대기
    ├─ 팝빌(POPBILL) → 세금계산서 역/정발행·국세청 전송
    ├─ 컬리 물류센터 → 입고 처리·반품 회수
    └─ 파트너 데이터 서비스 → 유료 리포트 (Basic 300만/월·Premium 900만/월)


[권한 구조 (Phase01)]
    권한 < 권한그룹 < 계정(이메일)
    └─ 기본권한그룹 ≥ 1 필수 + 특수권한그룹(선택)
    └─ old-escm + PP 이중 등록 필요 (Phase02 SSO okta 연동 시 단일화 예정)
```
