# 반품영역 사용자 기반 시나리오 정책서

> 출처: return_policy.md(파트너사용/임직원용 가이드) 재구성
> 관점: 사용자 유형(파트너사·컬리 임직원·시스템)별 행동·시나리오 중심

---

## 0. 사용자 유형 정의

| 유형 | 정의 | 주요 권한 |
|------|------|----------|
| **파트너사용자(공급사)** | 컬리에서 반품 요청을 받은 거래처 담당자. 파트너포털 공급사 권한으로 로그인 | 공급사확인대기 조회·확인완료·반품 거래명세서 출력·회수 진행 |
| **컬리 임직원 - 반품 담당자** | 반품 등록·승인 담당 (별도 이용가이드 참조 - §10 정책 확인 필요) | 반품 등록·승인 |
| **시스템(SYSTEM)** | 자동·배치 처리 주체 | 미응답 4일 자동 전환·회수지연 2회 폐기·반품 알림톡 발송·수거준비 전환 |

## 1. 반품 단계별 사용자 액션 매트릭스

| 처리 상태 | 파트너사용자 가능 행동 | 컬리 임직원 가능 행동 | 시스템 처리 |
|---------|------------------|------------------|----------|
| 공급사확인대기 | 회수방문일·연락처 입력 → [확인완료] | 검토 | 미응답 4일 시 자동 전환 |
| 회수대기 | 반품 거래명세서 출력·회수 진행 | 검토 | 일배치로 수거준비 전환 |
| 회수지연(1회) | 회수 재진행 | 검토 | 알림톡 발송 |
| 폐기 | 회수 종료 | 검토 | 회수지연 2회 시 자동 폐기, **비용 파트너사 부담** |

## 2. 파트너사용자(공급사) 시나리오

### 2-1. [정상] 반품 알림톡 수신 → 공급사확인대기 조회

| 항목 | 내용 |
|------|------|
| 진입 경로 | 카카오 알림톡 수신 → 파트너포털 로그인 → 반품관리 > 공급사확인대기 |
| 사용자 행동 | 반품 요청 상품·수량 조회 |
| 시스템 처리 | 본인 공급사 반품건만 노출 |
| 정책 근거 | §1, §5 |

### 2-2. [정상] 확인완료 (회수방문일·연락처 입력)

| 항목 | 내용 |
|------|------|
| 진입 경로 | 공급사확인대기 > 반품건 상세 |
| 사용자 행동 | 1. 회수방문일 입력<br>2. 회수 담당자 연락처 입력<br>3. [확인완료] |
| 시스템 처리 | [공급사확인대기] → [회수대기] 전이 |
| 정책 근거 | §1, §2 |

### 2-3. [정상] 반품 거래명세서 출력

| 항목 | 내용 |
|------|------|
| 진입 경로 | 회수대기 > 반품건 상세 |
| 사용자 행동 | [반품 거래명세서 출력] |
| 시스템 처리 | 거래명세서 PDF 다운로드 |
| 정책 근거 | §2 |

### 2-4. [정상] 회수 진행

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 입력한 회수방문일에 회수 차량 방문하여 상품 수거 |
| 시스템 처리 | 일배치로 수거준비 → 수거완료 흐름 진행 |
| 정책 근거 | §2, §4 |

### 2-5. [예외] 회수지연 1회

| 항목 | 내용 |
|------|------|
| 발생 조건 | 입력한 회수방문일에 회수 미진행 |
| 시스템 처리 | [회수지연(1회)] 전이, 공급사 담당자에게 알림톡 발송 |
| 후속 행동 | 회수 재진행 필요 |
| 정책 근거 | §1, §5 |

### 2-6. [예외] 회수지연 2회 → 자동 폐기 (비용 부담)

| 항목 | 내용 |
|------|------|
| 발생 조건 | 회수지연 1회 후 추가 지연 |
| 시스템 처리 | 자동 [폐기] 처리 |
| **비용** | **파트너사 부담** |
| 정책 근거 | §1, §3 |

### 2-7. [예외] 미응답 4일 → 자동 회수대기

| 항목 | 내용 |
|------|------|
| 발생 조건 | [공급사확인대기] 상태에서 4일간 [확인완료] 미클릭 |
| 시스템 처리 | 자동으로 [회수대기] 전환 |
| 영향 | 회수방문일·연락처 미입력 상태로 회수 흐름 진입 → 회수 차량 방문 정보 부재 위험 |
| 정책 근거 | §1, §3 |

### 2-8. [예외] 회수방문일·연락처 미입력으로 [확인완료] 시도

| 케이스 | 시스템 처리 |
|-------|-----------|
| 회수방문일 누락 | 차단 (필수 입력) |
| 회수 담당자 연락처 누락 | 차단 (필수 입력) |

## 3. 컬리 임직원 시나리오 - 반품 담당자

> ⚠️ Admin 반품 등록·승인 상세 흐름은 별도 이용가이드 참조 (§10 정책 확인 필요 §6 - ①)

### 3-1. [정상] 반품 등록

| 항목 | 내용 |
|------|------|
| 진입 경로 | Admin > 반품관리 > 반품 등록 |
| 사용자 행동 | 반품 대상 상품·수량 등록 |
| 시스템 처리 | 공급사에 반품 알림톡 발송, [공급사확인대기] 생성 |
| 정책 근거 | 별도 가이드 (확인 필요) |

### 3-2. [정상] 반품 승인

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 반품 등록 건 검토 후 [승인] |
| 시스템 처리 | 반품 진행 흐름 진입 |
| 정책 근거 | 별도 가이드 (확인 필요) |

### 3-3. [정상] 반품 진행 모니터링

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 공급사확인대기·회수대기·회수지연·폐기 상태 모니터링 |
| 시스템 처리 | 상태별 카운트·필터 제공 |
| 정책 근거 | §1 |

## 4. 시스템(자동·배치) 시나리오

### 4-1. 반품 수거준비 전환 배치

| 항목 | 내용 |
|------|------|
| 실행 주기 | 일배치 |
| 처리 내용 | 반품 상태 → 수거준비 상태 전환 |
| 정책 근거 | §4 |

### 4-2. 미응답 4일 자동 회수대기 전환

| 항목 | 내용 |
|------|------|
| 트리거 | [공급사확인대기] 상태 + 4일 경과 |
| 처리 내용 | [회수대기] 자동 전환 |
| 정책 근거 | §1, §3 |

### 4-3. 회수지연 1회 → 알림톡 발송

| 항목 | 내용 |
|------|------|
| 트리거 | 회수방문일 경과 + 회수 미진행 |
| 수신자 | 공급사 담당자 |
| 채널 | 카카오 알림톡 |
| 정책 근거 | §1, §5 |

### 4-4. 회수지연 2회 → 자동 폐기

| 항목 | 내용 |
|------|------|
| 트리거 | 회수지연 1회 이후 추가 지연 |
| 처리 내용 | [폐기] 자동 처리, 비용 파트너사 부담 |
| 정책 근거 | §1, §3 |

### 4-5. 반품 요청 알림톡 발송

| 항목 | 내용 |
|------|------|
| 트리거 | 컬리가 반품 등록 |
| 수신자 | 공급사 담당자 |
| 채널 | 카카오 알림톡 |
| 정책 근거 | §5 |

## 5. 사용자 시나리오 - 예외 케이스 모음

### 5-1. 권한 없는 메뉴 접근

| 케이스 | 시스템 처리 |
|-------|-----------|
| 공급사가 타 공급사 반품건 조회 | 자사 데이터만 응답 |
| 공급사가 반품 등록·승인 시도 | 차단 (컬리 전용) |

### 5-2. 입력값 검증

| 시도 | 차단 |
|------|------|
| 회수방문일 미입력 [확인완료] | 차단 |
| 회수 담당자 연락처 미입력 [확인완료] | 차단 |
| 과거 일자를 회수방문일로 입력 | 정책 확인 필요 (§6) |

### 5-3. 시간·일수 경계값 케이스

| 경계 | 검증 포인트 |
|------|-----------|
| 미응답 3일 23:59 vs 4일 00:00 | 자동 회수대기 전환 트리거 |
| 회수방문일 당일 22:59 vs 익일 00:00 | 회수지연 카운트 시작 시점 |
| 회수지연 1회 vs 2회 | 폐기 자동 전환 경계 |

### 5-4. 비용 부담 분기

| 케이스 | 비용 부담 |
|-------|---------|
| 정상 회수 완료 | - (정책상 명시 없음) |
| 회수지연 2회 폐기 | **파트너사 부담** |
| 정책 근거 | §3 |

## 6. 정책 확인 필요 항목

| # | 항목 | 사유 |
|---|------|------|
| 1 | Admin 반품 등록·승인 상세 | 별도 이용가이드 링크 참조 (§6) |
| 2 | 회수방문일 입력 가능 범위 | 과거·당일·미래 어디까지 가능한지 미명시 |
| 3 | 회수지연 카운트 시점 | 회수방문일 당일 24시 기준? 익일 카운트? |
| 4 | 미응답 4일 자동 전환 후 회수 흐름 | 회수방문일·연락처 부재 상태 처리 방법 |
| 5 | 정상 회수 시 비용 부담 주체 | 폐기 외 일반 회수 비용 정책 미명시 |

## 8. 반품 처리 ASCII 흐름도

```
[컬리: 반품 등록]
    │
    ▼ 알림톡 발송 (공급사)
[공급사확인대기]
    │
    ├─ 공급사 [확인완료] (회수방문일·연락처 입력) → [회수대기]
    └─ 미응답 4일 ──────────────────────────────────→ [회수대기] (자동)
                                                          │
                                                          ▼ 거래명세서 출력
                                                          ▼ 회수 진행
                                                          │
              ┌───────────────────────────────────────────┘
              │
              ├─ 정상 회수 → 수거준비 → 수거완료
              │
              └─ 회수 미진행
                     │
                     ▼ 1회
                [회수지연(1회)] ── 알림톡 발송 (공급사) ──→ 재시도
                     │
                     ▼ 2회
                [폐기] (자동, 비용 파트너사 부담)
```
