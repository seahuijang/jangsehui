## 1. 발주서 상태 전이 (파트너사용)

| 현재 상태 | 트리거 | 전이 상태 | 체크포인트 |
|---------|-------|---------|----------|
| 발주생성 | 추천일자 이후 + 변경 없음 입력 | 발주확정 | 즉시 확정 |

## 2. 발주관리 기본 정책

- 유통기한(소비기한) 유효성: 추천일자와 비교 → 발주확정/컬리승인대기/확정불가 분기
- 발주수량 변경: [변경] 버튼으로만 가능
- 엑셀 일괄 입력: 양식 1~2행 변경 불가, 최대 200건, 중복 마스터코드 → 마지막 덮어쓰기
- 발주확정 후 변경 마감: 입고예정일 D-1 오후 5시 (축산물 이력번호: D-1 23:59:59)
- 출고방법 변경 제약: 입고대행 발주 → '택배' 변경 불가
- 파렛트 수: 밀크런픽업 상품 존재 시 필수 입력

## 4. 발주관리 V1 정책

### 4-1. 과발주 승인 기준 (2024/09)

| 발주구분 | 과발주 기준 | 승인자 |
|---------|---------|------|
| 일반발주 | 판매예상일수 N일 이상 + 누적 공급가 합계 N원(50M) + 예외 카테고리 N원 | SCM |
| 행사(일특/라방) | 동일 + 승인자 공유사항 필수입력 | SCM & MD |
| 행사(기획/특가) | 동일 + 6개월 할인기간 일평균 판매수량 초과 | SCM |
| 신상품 | 1BOX 또는 10EA 초과 | MD |
| MD요청 | 동일 + 품의서 코드 필수 | SCM (누적 초과 시 SCM & MD) |
| 긴급발주 | 누적 공급가 20M (예외 카테고리 없음) | SCM |

**과발주 승인 생략 공급사**: 넥스트키친, 농협용인지점, CJ제일제당, 풀무원식품, 록야, 매일유업, 하림, 케이디글로벌, 엘지생활건강, 팜스코 (11개사)

### 4-2. 판매예상일수/발주수량 계산 공식

- **판매예상일수** = {(기초재고+총입고예정수량-(발주일~입고예정일 평균판매수량))+발주수량*입수} / 평균판매수량
- **발주수량** = [판매예상일수*평균판매수량-{기초재고+총입고예정수량-(발주일~입고예정일 평균판매수량)}] / 입수

### 4-3. 과발주 검토 팝업

- 최근 2주 평균 발주량 대비 300% 이상 → 과발주 팝업
- 입고예정수량 포함 판매예상일수 50일 이상 → 과발주 팝업
- 2주 평균 발주수량 배치: 새벽 00:40, 공급상태 '발주가능/견적서대기' 대상

### 4-4. 발주취소 사유

| 취소 사유 | 조건 |
|---------|------|
| 공급사 재고부족 | 발주생성/공급사승인대기에서 입고예정일 24시 미확정 |
| 재발주 | 재발주 버튼으로 취소 |
| 기타 | 발주취소 버튼 직접 |
| 자동 | 입고예정일 경과 미확정 → 다음날 00시 자동 |
| RMS 입고 완료 건 | 취소 불가 |

### 4-5. 구글시트 연동 자동발주 (서비스 일시중단)

- 배치: 일 3회 (10시/12시/14시) → 24/10/14 기준 14시만 수행
- 발주구분: '자동발주'로 생성
- 중복발주: 동일 ROW 2건 이상 시 판단
- 서비스 상태: **일시중단**

## 5. 발주관리 V2 정책 (Beta, 2026-03-09 오픈)

### 5-1. V1 대비 핵심 차이

| 항목 | V1 | V2 |
|------|------|------|
| 발주 단위 | 발주그룹 → 발주서 | 발주계획 → 발주상품 |
| 과발주 검사취소 | 불가 | 가능 (발주자만) |
| 거래명세서 | 자동 생성 | 병합/해제 기능 추가 |
| 상태값 | 6개 | 12개 |
| BigQuery | purchase_order_* | neo_purchase_order_* |
| V1↔V2 조회 | 상호 조회 불가 | 상호 조회 불가 |

### 5-2. V2 발주상품 12개 상태

1. 발주예정 (SCHEDULED)
2. 일CAPA 확인필요 (REQUIRED_CHECK_CAPA)
3. 일CAPA 검사완료 (FINISHED_CHECK_CAPA)
4. 과발주 확인필요 (REQUIRED_CHECK_EXCESS)
5. 과발주 승인대기 (WAITING_APPROVAL_EXCESS)
6. 과발주 반려 (REJECT_EXCESS)
7. 발주생성 (CREATED)
8. 컬리 승인대기 (WAITING_APPROVAL_KURLY)
9. 공급사 승인대기 (WAITING_APPROVAL_SUPPLIER)
10. 발주확정 (CONFIRMED)
11. 발주확정_명세서생성완료 (ISSUED)
12. 취소 (CANCELED)

### 5-3. 발주정보 변경 시 상태 롤백

| 변경 항목 | 롤백 상태 |
|---------|---------|
| 입고예정일/입고지 변경 | [발주예정] (공급사 미노출) |
| 출고방법 택배↔비택배 변경 | [발주예정] (공급사 미노출) |
| 발주구분 변경 | [발주예정] (공급사 미노출) |
| 발주수량(판매예상일수) 감량 | [공급사승인대기] |

### 5-4. 거래명세서 분리/병합 기준

- **분리 기준**: 공급사/상세입고지/경유여부/입고예정일/입고예약시간/발주구분/출고유형/출고방법/재고증량제외여부
- **병합 기준**: 마스터코드+소비기한+제조일자+단위/입수+과세구분+공급단가+증정품여부+판매방법+중량 동일

### 5-5. 컬리 승인대기 조건

1. 공급사 발주수량(감량) 변경
2. 경유센터 '입고대행' → '경유안함' 변경 (역방향 불가)
3. 출고방법 변경 (택배↔비택배, CAPA 잔여 충분=가능/부족=불가)
4. 소비기한/제조일자 임박한 일자 입력

## 6. 유통기한/제조일자 유효성 계산 로직

### 6-1. 입고기준율

| 유통기간(잔여) | 입고 기준율% |
|---------|---------|
| ~60일 | 70% 이상 |
| 61~179일 | 60% 이상 |
| 180일 이상 | 50% 이상 |

### 6-2. 1차 비교 공식

- **소비기한**: 입고예정일 + 최대[올림(유통가능일수x입고기준율) - 판매마감일수 vs 올림(판매가능일수x30%)] + 판매마감일수
- **제조일자**: 입고예정일 - {판매가능일수 - 최대[올림(유통가능일수x입고기준율) - 판매마감일수 vs 올림(판매가능일수x30%)]}

### 6-3. 승인대기 변경 로직

**소비기한(유통기한)**:
- 발주확정: 1차 비교 기준일자와 같거나 크게 입력
- 컬리 승인대기: (입고예정일+판매마감일수) 이후이나 추천일자 미만
- 발주확정불가: (입고예정일+판매마감일수) 미만

**제조일자**:
- 발주확정: 추천일자와 같거나 이후 입력
- 컬리 승인대기: (입고예정일-판매가능일수)와 같거나 크게 입력하고 추천일자 미만
- 발주확정불가: (입고예정일-판매가능일수) 미만

## 7. escm-purchase-silo 심화 정책

### 7-1. 발주 유형 및 과발주 검사

| 유형명 | 설명 | 과발주 검사 항목 |
|------|------|------|
| 일반발주 | 일반 발주 | 판매예상일수·누적금액 |
| 행사(일특/라방) | 일특·라방 행사 발주 | 판매예상일수·누적금액 |
| 행사(기획/특가) | 기획·특가 행사 발주 | 판매예상일수·누적금액·**할인기간 평판수량** |
| 신상품 | 신규 상품 발주 | 판매예상일수·**신상품 최대 발주수량** |
| MD요청 | MD 요청 발주 | 판매예상일수·누적금액·**MD요청 승인** |
| 공급사요청 | 공급사 요청 발주 | 판매예상일수·누적금액 |
| 자동발주 | 시스템 자동 생성 발주 | 판매예상일수·누적금액 |
| 긴급발주 | 긴급 처리 발주 | 판매예상일수·**긴급 누적금액** (일반 누적금액 검사 제외) |

- 긴급발주 제외, 나머지 모든 발주유형은 일CAPA 검사 수행
- 일CAPA 검사 진입 조건: **발주예정** 상태

### 7-2. 거래명세서 코드 생성 규칙

- 형식: `K[YYMMDD]_[7자리 랜덤 영숫자 대문자]`
- 예: `K260331_AB12C3D`
- 병합 기준 키: 공급사코드 + 입고예정일 + 입고시간 + 클러스터센터 + 풀필먼트센터 + 도크 + 경유센터 + 발주구분 + 출고방법 + 출고유형 + 재고증량제외여부 + 입고관리시스템 운영시간ID

### 7-3. 발주계획상품 전체 상태 흐름

```
[발주예정]
    │
    ▼ 일CAPA 예약 요청 (입고관리시스템 연동)
    ├─ 예약 성공 → [일CAPA 검사완료]
    └─ 예약 실패 → [일CAPA 확인필요]
                         │
                         └─ 수동 재시도 → [일CAPA 검사완료]

[일CAPA 검사완료]
    │
    ▼ 과발주 검사
    ├─ 과발주 아님 → [발주생성]
    └─ 과발주 감지 → [과발주 확인필요]
                         │
                         ▼ 승인 요청
                     [과발주 승인대기]
                         │
                         ├─ 승인 → [발주생성]
                         └─ 반려 → [과발주 반려] → 취소 처리

[발주생성]
    │
    ▼ 공급사에 알림톡 발송
    ├─ 공급사 승인 대기 → [공급사 승인대기]
    └─ 컬리 내부 검토 → [컬리 승인대기]

[공급사 승인대기]
    │
    ├─ 공급사 확정 → [발주확정]
    └─ 컬리 검토 → [컬리 승인대기]

[컬리 승인대기]
    │
    └─ 컬리 승인 → [발주확정]

[발주확정]
    │
    └─ 거래명세서 생성 → [명세서생성완료]
```

### 7-4. 취소 가능 상태

- 발주예정, 일CAPA 확인필요, 일CAPA 검사완료
- 과발주 확인필요, 과발주 승인대기, 과발주 반려
- 공급사 승인대기, 컬리 승인대기, 발주생성
> **발주확정, 명세서생성완료 상태는 취소 불가**

### 7-5. 거래명세서/발주그룹/과발주승인 상태 흐름

**거래명세서**: `[생성완료] → [폐기]`

**발주그룹**: `[발주대기] → [승인요청] → [완료]` / `[발주대기]/[승인요청] → [취소]`

**과발주 승인**: `[확인필요] → [승인대기] → [승인완료]/[승인반려]/[취소]` (유형: 사전승인만)

### 7-6. 주요 제약 조건

#### 수량 제약

| 조건 | 규칙 |
|------|------|
| 수량 변경 방향 | **감량만 가능** (증량은 발주예정, 일CAPA 확인필요 상태에서만 허용) |
| 입고관리시스템 CAPA 수정 | 감소 방향 수정만 허용 |
| 오류 메시지 | "발주 수량은 감량만 가능합니다. (기존수량: {n}, 수정수량: {n})" |

#### 과발주 승인 제약 — 발주유형별 검사 항목

| 발주유형 | 판매예상일수 | 누적공급가 | 할인평판수량 | 신상품수량 | MD요청 |

| 타 공급사 상품 포함 확정 | "%s 공급사의 상품만 확정할 수 있습니다." | 다른 공급사 상품 포함 시 |
| 수정 불가 상태 수정 시도 | "수정할 수 없는 상품이 포함되어 있습니다." | 수정 불가 상태 상품 수정 시도 |

#### 입고관리시스템(RMS) 연동 예외 처리

| 상황 | 처리 방식 |
|------|---------|
| 입고관리시스템 서버 오류 | 발주계획상품 상태 변경 없이 유지 |
| CAPA 예약 성공 (예약키 존재) | **일CAPA 검사완료** 상태로 전환 + 예약키 저장 |
| CAPA 예약 실패 (예약키 없음) | **일CAPA 확인필요** 상태로 전환 |
| 더블 부킹 감지 | 예약키 원복, 후보키 → 예약키로 덮어쓰기 |

#### 자동 취소 배치 예외 처리

- 취소 대상: 취소 가능 상태 + 기준일 이전 미처리 건
- 취소자: 시스템(SYSTEM) 자동 처리 표기
- 청크 단위: 1,000건 분할 처리
- 과발주 승인 취소 가능 상태: 과발주 확인필요, 과발주 승인대기, 과발주 반려
