# 입고영역 사용자 기반 시나리오 정책서

> 관점: 사용자 유형(파트너사·컬리 임직원·시스템)별 행동·시나리오 중심

## 0. 사용자 유형 정의

| 유형 | 정의 | 주요 권한 |
|------|------|----------|
| **파트너사용자(공급사)** | 컬리에 상품을 입고시키는 거래처의 담당자. 파트너포털 공급사 권한으로 로그인 | 입고예정 조회·입고확정·클레임 신청 |
| **컬리 임직원 - 입고 담당자(IM/OB/LC)** | 입고 처리·검수·재고 반영 담당 | 입고 처리, 입고확정보류 수량 입력 |
| **컬리 임직원 - 팀장 이상** | 발주·반품 승인 권한자 | 발주 승인, 반품 승인 |
| **컬리 임직원 - MD/AMD/발주자** | 클레임 수신·검토 담당 | 클레임 이메일 수신, 검토 |
| **컬리 임직원 - MASTER** | 전 메뉴 접근 | 전 기능 |
| **시스템(SYSTEM)** | 자동·배치 처리 주체 | 입고 결과 일괄 반영, 자동 입고확정, 만료 처리, 알림 발송 |
| **외부 - RMS(입고관리시스템)** | 입고 이슈 상세 입력 시스템 | RMS 화면에서 이슈 입력 |

## 1. 입고 단계별 사용자 액션 매트릭스

| 처리 상태 | 파트너사용자 가능 행동 | 컬리 임직원 가능 행동 | 시스템 처리 |
|---------|------------------|------------------|----------|
| 입고예정 | 조회만 | 조회·발주 변경(D-1 17시 전) | 입고당일 23시 일괄 반영 |
| 입고 대기 | 조회 | 처리 시작·취소 | 알림 발송 |
| 입고 진행 중 | 조회 | 검수 완료 처리 | - |
| 검수 완료 | 조회 | 재고 반영 | - |
| 입고확정대기 | [입고확정] 클릭 | 입고확정보류 수량 입력 | 익일 00:10 결과 반영 |
| 입고확정 | 클레임 신청(3일 이내) | 클레임 검토 | +3일 미처리 시 자동확정 |
| 재고 반영 완료 | 반품 요청 | 반품 처리 | 정산 기준일 = 입고일 |

## 2. 파트너사용자(공급사) 시나리오

### 2-1. [정상] 입고예정 조회

| 항목 | 내용 |
|------|------|
| 진입 경로 | 파트너포털 > 입고관리 > 입고예정 |
| 사용자 행동 | 입고예정일·상품·수량 확인 |
| 시스템 처리 | 입고당일 23시 이후 자동으로 입고현황/입고확정 메뉴로 반영 |
| 정책 근거 | §1, §2 |

### 2-2. [정상] 입고확정 클릭

| 항목 | 내용 |
|------|------|
| 진입 경로 | 입고관리 > 입고확정대기 |
| 사용자 행동 | 1. 입고 결과 확인<br>2. [입고확정] 클릭 |
| 시스템 처리 | [입고확정대기] → [입고확정] 전이 |
| 정책 근거 | §1 |

### 2-3. [정상] 클레임 신청 (입고확정 +3일 이내)

| 항목 | 내용 |
|------|------|
| 진입 경로 | 입고관리 > 입고확정 > 클레임 신청 |
| 사용자 행동 | 1. 거래명세서 + 운송장 첨부 (증빙 2종 필수)<br>2. 사유 작성<br>3. 신청 |
| 시스템 처리 | 상품 담당 MD·AMD·발주자에게 이메일 발송 |
| 가능 기간 | 입고확정일로부터 **3일 이내** |
| 이의제기 채널 | coqa@kurlycorp.com |
| 정책 근거 | §1, §2, §8 |

### 2-4. [예외] 클레임 신청 마감 경과

| 케이스 | 시스템 처리 |
|-------|-----------|
| 입고확정 +3일 초과 후 클레임 시도 | 신청 차단 |
| 증빙 2종(거래명세서·운송장) 중 하나라도 누락 | 신청 차단 |

### 2-5. [예외] V2 발주 건 입고예정 미노출

| 케이스 | 시스템 처리 |
|-------|-----------|
| V2 발주확정_명세서생성완료 건 [입고예정] 메뉴 조회 | 미노출 → 입고현황/입고확정에서만 확인 가능 |
| 정책 근거 | §8 (정책 확인 필요 §11 - ①) |

### 2-6. [예외] 자동 입고확정 (3일 미처리)

| 케이스 | 시스템 처리 |
|-------|-----------|
| 공급사가 입고확정대기에서 +3일간 [입고확정] 미클릭 | 시스템이 자동으로 [입고확정] 처리 |
| 영향 | 자동 확정 후 클레임 가능 기간 카운트 시작 |
| 정책 근거 | §1, §2 |

## 3. 컬리 임직원 시나리오 - 입고 담당자(IM/OB/LC)

### 3-1. [정상] 입고 처리 시작

| 항목 | 내용 |
|------|------|
| 진입 경로 | Admin > 입고관리 > 입고 대기 목록 |
| 사용자 행동 | 1. 입고 건 선택<br>2. 처리 시작 |
| 시스템 처리 | [입고 대기] → [입고 진행 중] |
| 정책 근거 | §3 |

### 3-2. [정상] 검수 완료 처리

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 검수 결과 입력 후 [검수 완료] |
| 시스템 처리 | [입고 진행 중] → [검수 완료] |
| 수량 제약 | 발주 수량의 **±10% 이내**만 허용, 0개·음수 불가 |
| 정책 근거 | §3, §4 |

### 3-3. [정상] 재고 반영

| 항목 | 내용 |
|------|------|
| 사용자 행동 | [재고 반영] 처리 |
| 시스템 처리 | [검수 완료] → [재고 반영 완료], 익일 00:10 일괄 결과 반영 |
| 정책 근거 | §2, §3 |

### 3-4. [정상] 입고확정보류 수량 입력 (컬리 담당자)

| 항목 | 내용 |
|------|------|
| 발생 조건 | 동일 상품 + 다른 공급단가 입고 |
| 사용자 행동 | [입력] 버튼으로 수량 직접 기입 |
| 시스템 처리 | [입고확정보류] → [입고확정대기] 전환 |
| 정책 근거 | §2, §8 |

### 3-5. [정상] 입고 이슈 상세 입력 (RMS)

| 항목 | 내용 |
|------|------|
| 진입 경로 | RMS(입고관리시스템) 화면 |
| 사용자 행동 | RMS에서 이슈 상세 직접 입력 |
| 시스템 처리 | **공급사에 메일·알림톡 별도 발송 없음** |
| 정책 근거 | §8 |

### 3-6. [예외] 입고 진행 중 취소 시도

| 케이스 | 시스템 처리 |
|-------|-----------|
| [입고 진행 중] 상태에서 취소 시도 | 차단 (취소는 [입고 대기] 상태에서만 가능) |
| 정책 근거 | §3 |

### 3-7. [예외] 검수 완료·재고 반영 후 수량 수정

| 케이스 | 시스템 처리 |
|-------|-----------|
| [검수 완료] 상태에서 수량 수정 | 차단 |
| [재고 반영 완료] 상태에서 취소·수정 | 차단 |
| 정책 근거 | §3 |

### 3-8. [예외] 수량 초과 입고 처리

| 케이스 | 시스템 처리 |
|-------|-----------|
| 발주 수량의 ±10% 초과 입고 입력 | 차단 |
| 0개 또는 음수 입고 입력 | 차단 |
| 정책 근거 | §4 |

### 3-9. [예외] 중복 입고 처리

| 케이스 | 시스템 처리 |
|-------|-----------|
| 동일 입고 번호로 두 번 처리 시도 | 두 번째 시도 차단 |
| 처리 중 상태에서 동시 요청 (Race Condition) | 하나만 처리, 나머지 차단 |
| 정책 근거 | §5 |

| 항목 | 내용 |

## 7. 사용자 시나리오 - 예외 케이스 모음

### 7-1. 권한 없는 메뉴 접근

| 케이스 | 시스템 처리 |
|-------|-----------|
| 입고 담당자 미만이 입고 처리 시도 | 차단 |
| 일반 직원이 발주·반품 승인 시도 | 차단 / 메뉴 미노출 |
| 공급사가 타 공급사 입고건 조회 | 자사 데이터만 응답 |

### 7-2. 상태별 차단 처리

| 시도 | 차단 메시지·결과 |
|------|---------------|
| [입고 진행 중] 상태 취소 | 차단 (취소는 [입고 대기]만) |
| [검수 완료] 수량 수정 | 차단 |
| [재고 반영 완료] 취소·수정 | 차단 |
| 만료된 발주 건 재처리 | 차단, 신규 발주 필요 |
| 입고확정 +3일 초과 클레임 | 차단 |

### 7-3. 입력값 검증

| 시도 | 차단 |
|------|------|
| 발주 수량 ±10% 초과 입고 | 차단 |
| 0개·음수 입고 | 차단 |
| 클레임 증빙 2종 중 하나 누락 | 차단 |
| 동일 입고 번호 중복 처리 | 차단 |

### 7-4. 자주 발생하는 결함 유형 (QA 중점 검증)

| 결함 유형 | 검증 포인트 |
|---------|-----------|
| 수량 초과 처리 허용 | ±10% 한도, 0/음수 한도 검증 |
| 상태 불일치 | 화면 표시 vs 실제 DB 상태 |
| 재고 미반영 | 입고 완료 후 재고 수량 갱신 |
| 중복 입고 허용 | 동일 입고 번호 중복 처리 차단 |
| 권한 오류 | 비권한자 승인·반품 차단 |
| 만료 건 재처리 | 만료 후 재활성화 차단 |

### 7-5. 시간·날짜 경계값 케이스

| 경계 | 검증 포인트 |
|------|-----------|
| 입고당일 22:59 vs 23:00 | 23시 이후 자동 반영 트리거 |
| 익일 00:09 vs 00:10 | 00:10 일괄 결과 반영 트리거 |
| 입고확정일 +3일 23:59 vs +4일 00:00 | 클레임 가능/자동확정 경계 |
| 입고예정일 +3일 vs +4일 | 자동 만료 경계 |

## 8. 정책 확인 필요 항목

| # | 항목 | 사유 |
|---|------|------|
