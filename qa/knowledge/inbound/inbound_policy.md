# 입고관리 정책서

> 출처: 파트너포털 이용가이드(파트너사용) v2025.03 + [컬리임직원] Partner Portal admin 이용가이드 v2026.02

---

## 1. 입고 상태 전이 (파트너사용)

| 현재 상태 | 트리거 | 전이 상태 |
|---------|-------|---------|
| 입고예정 | 입고당일 23시 이후 시스템 반영 | 입고현황/입고확정 |
| 입고확정대기 | 파트너 [입고확정] 클릭 | 입고확정 |
| 입고확정대기 | 미처리 +3일 경과 | 입고확정(자동) |
| 입고확정 | 확정일로부터 3일 이내 | 클레임 신청 가능 |
| 입고확정 | 확정일로부터 3일 초과 | 클레임 불가 |

---

## 2. 입고관리 기본 정책

- 입고결과 반영 시점: 익일 00:10 일괄
- 정산 기준일: "입고일" (입고예정일 아님)
- 입고확정보류 생성 조건: 동일 상품 + 다른 공급단가 입고
- 자동 입고확정: 입고일 +3일 미처리 시
- 클레임: 입고확정일 3일 이내, 증빙 2종 필수 (거래명세서+운송장)
- 이의제기: coqa@kurlycorp.com

---

## 3. 입고 처리 상태 전이 (상세)

```
입고 대기 → 입고 진행 중 → 검수 완료 → 재고 반영 완료
    ↓
  취소 완료 (입고 진행 전에만 가능)
```

### 상태별 허용 행동

| 상태 | 허용 행동 | 불가 행동 |
|------|---------|---------|
| 입고 대기 | 처리 시작, 취소 | 재고 반영 |
| 입고 진행 중 | 검수 완료 처리 | 취소 |
| 검수 완료 | 재고 반영 | 수량 수정 |
| 재고 반영 완료 | 반품 요청 | 취소, 수정 |

---

## 4. 수량 정책

- 입고 수량은 발주 수량의 ±10% 이내만 허용
- 0개 또는 음수 입고 불가
- 부분 입고 허용 여부는 품목별 설정에 따름

---

## 5. 중복 처리 방지

- 동일 입고 번호로 중복 처리 불가
- 처리 중 상태에서 동시 요청 시 하나만 처리

---

## 6. 권한 정책

- 입고 처리: 입고 담당자 이상
- 발주 승인: 팀장 이상
- 반품 승인: 팀장 이상

---

## 7. 마감 정책

- 입고 예정일 기준 D+3일 이후 미처리 건 자동 만료
- 만료된 건은 재활성화 불가, 신규 발주 필요

---

## 8. Admin 입고관리 전용 정책

| 항목 | 정책 |
|------|------|
| 입고 이슈 상세 | RMS에서 입력, 공급사에 메일/알림톡 별도 발송 안됨 |
| 입고확정보류 처리 | 컬리 담당자가 '입력' 버튼으로 수량 직접 기입 → 입고확정대기 전환 |
| 클레임 이메일 수신 | 상품 담당MD, AMD, 발주자 이메일 |
| V2 입고예정 | V2 발주확정_명세서생성완료 건은 [입고예정]에 미노출 |
| 입고확정대기 알림톡 | 공급사별 건수 안내, 3일 후 자동확정 경고 포함 |

## 10. QA 시 자주 발생하는 결함 유형

1. **수량 초과 처리 허용** — 정책 한도를 초과해도 입고가 완료되는 현상
2. **상태 불일치** — 화면 표시 상태와 실제 처리 상태가 다른 현상
3. **재고 미반영** — 입고 완료 후 재고 수량이 업데이트되지 않는 현상
4. **중복 입고 허용** — 동일 건에 대해 두 번 처리가 가능한 현상
5. **권한 오류** — 권한 없는 사용자가 승인/반품 처리 가능한 현상
6. **만료 건 재처리** — 만료된 발주 건이 다시 처리 가능한 현상
