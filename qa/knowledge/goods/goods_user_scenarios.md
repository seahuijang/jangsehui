> 관점: 사용자 유형(파트너사·컬리 임직원·시스템)별 행동·시나리오 중심

## 0. 사용자 유형 정의

| 유형 | 정의 | 주요 권한 |
|------|------|----------|
| **파트너사용자(공급사)** | 컬리에 상품을 공급하는 거래처의 담당자. 파트너포털 공급사 권한으로 로그인 | 신규 상품 공급정보 입력·등록 요청, 견적서 등록 |
| **컬리 임직원 - MD/AMD** | 상품 기획 담당. 상품 마스터·공급정보 전권 | 상품등록·마스터정보 수정·입고지·센터항목·재고운영·공급정보 수정·다운 전권 |
| **컬리 임직원 - CO (SCM/QA)** | 공급망/품질 담당. 멀티벤더 및 공급사단위 운영 | MD/AMD 권한 + (공급사단위) 센터항목 정보 수정·권한 부여 계정에 한해 공급정보 수정 |
| **컬리 임직원 - IM** | 입고·센터·재고 운영 담당 | 입고지·센터항목·재고운영 수정·다운 |
| **컬리 임직원 - RM** | 재고 운영 담당 | 재고운영 정보만 수정·다운 |
| **시스템(SYSTEM)** | 자동 처리 주체 | 견적서 유효기간 만료 처리 |

## 2. 파트너사용자(공급사) 시나리오

### 2-1. [정상] 신규 상품 등록 요청

| 항목 | 내용 |
|------|------|
| 진입 경로 | 파트너포털 > 상품관리 > 신규 상품 등록 |
| 사용자 행동 | 1. 마스터·공급 정보 입력<br>2. 한글표시사항 이미지 업로드<br>3. [등록] 버튼 |
| 시스템 처리 | [상품생성] → [MD승인대기] 전이, MD에게 검토 요청 |
| 정책 근거 | §1 공급상태 전이 |

### 2-2. [정상] 한글표시사항 이미지 업로드

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 한글표시사항 이미지 첨부 |
| 시스템 제약 | PNG/JPEG만 허용, 10MB 이하 |
| 차단 시나리오 | 다른 확장자(JPG·GIF·PDF 등) / 10MB 초과 → 업로드 차단 |
| 정책 근거 | §2 |

### 2-3. [정상] 견적서 등록

| 항목 | 내용 |
|------|------|
| 진입 경로 | 계약관리 > 견적서 등록 |
| 사용자 행동 | 1. 공급가(낱개단위, VAT 제외) 입력<br>2. 견적서 구분 선택(일반/이벤트)<br>3. 유효기간 설정<br>4. [등록] |
| 가능 상태 | 견적서대기·발주가능 상태에서만 등록 가능 |
| 시스템 처리 | 컬리 승인 후 [발주가능] 전이 |
| 정책 근거 | §3 |

### 2-4. [정상] 견적서 유효기간 겹침 시 동일 구분 등록

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 동일 구분(일반/이벤트) 견적서를 유효기간 겹치게 등록 |
| 시스템 처리 | **최신 등록 공급가만 유효** 처리, 이전 견적서 무효화 |
| 정책 근거 | §3 |

### 2-5. [예외] 한글표시사항 이미지 형식·용량 위반

| 케이스 | 시스템 처리 |
|-------|-----------|
| 확장자 불일치 (JPG·GIF·PDF·HEIC 등) | 업로드 차단 |
| 10MB 초과 | 업로드 차단 |

### 2-6. [예외] 컬리냉매제 입력 (역직관적 정책)

| 입력 값 | 의미 |
|---------|------|
| '필요없음' | 냉매제 **포함** (공급사가 냉매제 동봉) |
| '필요' | 냉매제 **미포함** (컬리가 냉매제 추가) |

> 표기와 실제 동작이 반대 → 입력 시 안내 문구 노출 권장 (정책 확인 필요)

### 2-7. [예외] 유의성질여부 (유리포함)

| 항목 | 내용 |
|------|------|
| 체크 조건 | 유리포함 상품만 |
| 미체크 케이스 | 유리 미포함 상품인데 체크 → 잘못된 포장·배송 처리 위험 |

### 2-8. [예외] 견적서 만료 후 재등록

| 항목 | 내용 |
|------|------|
| 트리거 | 유효기간 만료 시 시스템 배치로 [기간만료] 처리 |
| 사용자 행동 | 재등록 + 컬리 재승인 필요 |
| 시스템 처리 | 견적서대기 → 발주가능 흐름 재진입 |
| 정책 근거 | §3 |

## 3. 컬리 임직원 시나리오 - MD/AMD

### 3-1. [정상] 신상품 등록 (단건)

| 항목 | 내용 |
|------|------|
| 진입 경로 | Admin > 상품관리 > 단건 등록 |
| 사용자 행동 | 1. 마스터·공급 정보 입력<br>2. [등록] |
| 시스템 처리 | 즉시 [상품생성] 후 운영 흐름 진입 |
| 정책 근거 | §4 |

### 3-2. [정상] 신상품 엑셀 등록 (양식 1번 - 상품&공급사 정보 등록)

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 양식 다운 → 다건 입력 → 업로드 |
| 시스템 처리 | 검증 통과 건만 등록, 오류 건은 사유와 함께 반환 |
| 정책 근거 | §5 엑셀 8종 - ① |

### 3-3. [정상] 멀티벤더 공급정보 추가 (양식 2번)

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 기존 상품에 공급사코드 추가 엑셀 업로드 |
| 시스템 처리 | 동일 마스터코드에 다수 공급사 매핑 |
| 정책 근거 | §5 - ② |

### 3-4. [정상] MD 승인

| 항목 | 내용 |
|------|------|
| 진입 경로 | 상품관리 > MD 승인대기 목록 |
| 사용자 행동 | 1. 공급사 등록 상품 검토<br>2. [승인] |
| 시스템 처리 | [MD승인대기] → [견적서대기] 전이 |
| 정책 근거 | §1 |

### 3-5. [정상] 견적서 승인

| 항목 | 내용 |
|------|------|
| 진입 경로 | 계약관리 > 견적서 승인대기 |
| 사용자 행동 | 1. 공급가·구분·유효기간 검토<br>2. [승인] |
| 시스템 처리 | [견적서대기] → [발주가능] 전이 |
| 정책 근거 | §1, §3 |

### 3-6. [정상] 마스터정보 수정 (상품 마스터정보 양식 - 양식 3번)

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 마스터 정보 수정 엑셀 업로드 |
| 시스템 처리 | **모든 CC에 동일 값으로 일괄 저장** |
| 주의 | CC별로 다르게 하려면 양식 5·8번 사용 |
| 정책 근거 | §5 |

### 3-7. [정상] 출하코드 입력

| 상품 유형 | 출하코드 입력 |
|---------|------------|
| 본품 | 마스터코드와 **동일** |
| 대품(증정품/이벤트) | **본품의 마스터코드** 입력 |
| 대체코드 | 본품에만 입력, **중복 불가** |
| 정책 근거 | §7 |

### 3-8. [예외] 대체코드 중복 입력

| 케이스 | 시스템 처리 |
|-------|-----------|
| 동일 대체코드 2건 이상 입력 | 저장 차단 |
| 대품에 대체코드 입력 | 저장 차단 (본품에만 가능) |

### 3-9. [예외] 마스터정보 양식 오용으로 전 CC 덮어쓰기

| 케이스 | 시스템 처리 |
|-------|-----------|
| CC별 다른 값을 마스터정보 양식에 입력 | 모든 CC에 동일 값 저장 → 데이터 손실 |
| 권장 동작 | 사전 안내 팝업 노출 (정책 확인 필요 §8 - ①) |

## 4. 컬리 임직원 시나리오 - CO (SCM/QA)

### 4-1. [정상] 공급사단위 센터항목 정보 수정 (양식 8번)

| 항목 | 내용 |
|------|------|
| 권한 | CO 전용 |
| 사용자 행동 | 공급사단위 센터항목 양식 다운 → 입력 → 업로드 |
| 시스템 처리 | 공급사 단위로 모든 상품 센터항목 일괄 적용 |
| 정책 근거 | §5 - ⑧ |

### 4-2. [정상] 공급정보 수정 (권한 부여 계정 한정)

| 항목 | 내용 |
|------|------|
| 권한 | CO 중 공급정보 수정 권한 부여된 계정만 |
| 사용자 행동 | 양식 7번 업로드 또는 단건 수정 |
| 시스템 처리 | 변경분 저장, 견적서 영향 시 재승인 필요 여부 판단 |
| 정책 근거 | §4, §5 - ⑦ |

### 4-3. [예외] 권한 미부여 CO 계정의 공급정보 수정 시도

| 케이스 | 시스템 처리 |
|-------|-----------|
| 권한 미부여 CO가 공급정보 수정 시도 | 차단 / 메뉴 미노출 |

## 5. 컬리 임직원 시나리오 - IM

### 5-1. [정상] 입고지정보 수정 (양식 4번)

| 항목 | 내용 |
|------|------|
| 진입 경로 | 상품관리 > 입고지정보 수정 |
| 사용자 행동 | 단건 수정 또는 양식 4번 엑셀 업로드 |
| 시스템 처리 | 입고지 변경 저장, 발주 가능 입고지 갱신 |
| 정책 근거 | §4, §5 - ④ |

### 5-2. [정상] 상품단위 센터항목 정보 수정 (양식 5번)

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 양식 5번에 CC별 유통가능일수·소비기한일수·판매가능일수·판매마감일수 입력 |
| 시스템 처리 | CC별 다른 값으로 저장 |
| 정책 근거 | §2, §5 - ⑤ |

### 5-3. [정상] 재고운영 정보 수정 (양식 6번)

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 양식 6번 업로드 |
| 시스템 처리 | 상세FC·재고 정책 갱신 |
| 정책 근거 | §5 - ⑥ |

### 5-4. [정상] 오더유형 설정

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 센터별 셋별·택배·대량 오더유형 설정 |
| 시스템 처리 | 변경 시 **재고운영정보 상세FC 초기화 → 재설정 필수** |
| 디폴트 노출 | 상품 재고정보 화면은 **2CC(김포)** 값 |
| 정책 근거 | §6 |

### 5-5. [예외] 오더유형 변경 후 상세FC 미재설정

| 케이스 | 시스템 처리 |
|-------|-----------|
| 변경 후 상세FC 재설정 누락 | 발주·입고 시 잘못된 FC 매핑으로 운영 오류 |
| 권장 동작 | 변경 시점에 재설정 안내 메시지 노출 (정책 확인 필요 §8 - ②) |

## 7. 시스템(자동) 시나리오

### 7-1. 견적서 유효기간 만료 처리

| 항목 | 내용 |
|------|------|
| 트리거 | 유효기간 종료 시점 도달 |
| 시스템 처리 | [발주가능] → [견적서대기]로 자동 전이, '기간만료' 표시 |

| 정책 근거 | §1, §3 |

### 7-2. 마스터정보 양식 업로드 시 전 CC 덮어쓰기

| 항목 | 내용 |
|------|------|
| 트리거 | 양식 3번(마스터정보) 또는 양식 7번(공급정보) 업로드 |
| 시스템 처리 | 입력 값을 **모든 CC에 동일 적용** |
| 위험 | CC별 다른 값을 의도한 경우 데이터 손실 |
| 정책 근거 | §5 |

## 8. 사용자 시나리오 - 예외 케이스 모음

### 8-1. 권한 없는 메뉴 접근

| 케이스 | 시스템 처리 |
|-------|-----------|
| IM이 상품등록·마스터정보·공급정보·상품&공급정보 다운 시도 | 메뉴 미노출 / 차단 |
| RM이 입고지·센터항목 수정 시도 | 메뉴 미노출 / 차단 |
| MD/AMD가 공급사단위 센터항목 수정(양식 8번) 시도 | 차단 (CO 전용) |
| 권한 미부여 CO가 공급정보 수정 시도 | 차단 |

### 8-2. 입력값·파일 검증

| 시도 | 차단 |
|------|------|

### 8-3. 정책 오해 위험 케이스

### 8-4. 엑셀 8종 양식 오용
