> 관점: 사용자 유형(파트너사·컬리 임직원·시스템·외부)별 행동·시나리오 중심

## 0. 사용자 유형 정의

| 유형 | 정의 | 주요 권한 |
|------|------|----------|
| **파트너사용자(공급사)** | 정산 승인·반려 및 계산서 발행 주체 | 정산 승인/반려·정발행 계산서 발행·증빙 첨부·정산 문의 등록·대금지급 조회 |
| **컬리 임직원 - CO(QA)** | 정산 운영 핵심 권한 | 정산 집계·매입 조정 대량 엑셀·정산 문의 답변·정산 취소·계산서 발행 취소·대금지급 산출 |
| **컬리 임직원 - CO부서** | 매입 조정 대량 권한 | 매입 조정 대량 엑셀(최대 1,000건) |
| **컬리 임직원 - 일반(MD/AMD/SCM 등)** | 정산 현황 조회 | 본인 담당 영역 조회 (정산 처리 권한 없음) |
| **시스템(SYSTEM)** | 자동·배치 처리 주체 | 역발행 자동 처리·국세청 익일 전송·이월 상계 매월 1일 마감·알림톡 발송 |
| **외부 - 팝빌(KURLYBILL_PU)** | 전자세금계산서 연동 시스템 | 발행/취소 처리·국세청 전송 중계 |

## 2. 정산 단계별 흐름 매트릭스

| 단계 | 주체 | 행동 | 결과 상태 |
|------|------|------|----------|
| ① 매입조정 | CO부서/CO(QA) | 정산포함 여부·금액 조정 | 집계여부 N |
| ② 정산집계 | CO(QA) | 집계기준일·세무구분·공급사·마감일 입력 | 집계완료 |
| ③ 정산현황 | CO(QA) | 승인요청 발송 | 승인요청 |
| ④ 공급사정산승인 | 공급사 | 승인 또는 반려(사유 256자) | 승인완료 / 승인반려 |
| ⑤ 발행대기 → 계산서발행 | 시스템(역발행) / 공급사(정발행) | 자동 또는 수동 | 계산서발행 |
| ⑥ 국세청전송 | 시스템 | 익일 자동 전송 | 국세청전송완료 |
| ⑦ 매입금액관리 | CO(QA) | 당월 매입금액 산출 (≤50개) | 매입금액 산출 |
| ⑧ 상계금액관리 | CO(QA) | 원천 상계·구매론·이월·마감변경 등록 | 상계금액 확정 |
| ⑨ 대금지급액계산 | CO(QA) | 회차별 최종지급금액 산출 | 지급예정 |
| ⑩ 대금지급현황 | 공급사 / CO(QA) | 조회 및 확인 | 최종지급 |
| ⑪ 정산문의 | 공급사(등록) / CO(QA)(답변) | Q&A | 답변완료 |

## 3. 파트너사용자(공급사) 시나리오

### 3-1. [정상] 정산 승인

| 항목 | 내용 |
|------|------|
| 진입 경로 | 파트너포털 > 정산관리 > 정산 현황 |
| 사용자 행동 | 1. 정산 내역 검토<br>2. [승인] |
| 시스템 처리 | 상태 [승인요청] → [승인완료] → [발행대기] → 자동 역발행 |
| 후속 처리 | 익일 국세청 자동 전송 |
| 정책 근거 | §1, §2 |

### 3-2. [정상] 정산 반려 (반려사유 필수 256자)

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 1. 반려사유 입력 (필수, 256자)<br>2. [반려] |
| 시스템 처리 | 상태 [승인요청] → [승인반려] |
| 정책 근거 | §1, §2 |

### 3-3. [정상] 정발행 계산서 발행 (증빙 첨부)

| 항목 | 내용 |
|------|------|
| 가능 조건 | 정발행 대상 정산 |
| 사용자 행동 | 1. 계산서 직접 발행 (팝빌 연동)<br>2. 증빙 첨부 |
| 시스템 처리 | [정발행대기] → [계산서발행] |
| 외부 연동 | 팝빌 KURLYBILL_PU |
| 정책 근거 | §2 |

### 3-4. [정상] 정산 문의 등록

| 항목 | 내용 |
|------|------|
| 진입 경로 | 정산관리 > 정산 문의 |
| 사용자 행동 | 1. 문의 내용 작성<br>2. 첨부파일(1개) 업로드<br>3. [등록] |
| 시스템 처리 | CO(QA) 답변 대기 |
| 제약 | 첨부파일 1개만, 개인정보 포함 업로드 금지 |
| 정책 근거 | §11 |

### 3-5. [정상] 대금지급 현황 조회

| 항목 | 내용 |
|------|------|
| 진입 경로 | 정산관리 > 공급사대금지급현황 |
| 사용자 행동 | 회차별 최종지급금액 조회 |
| 미노출 항목 | **구매론발행금액 / 마감변경 / 컬리조정** (공급사 미노출) |
| 정책 근거 | §10 |

### 3-6. [예외] 반려사유 미입력 [반려]

| 케이스 | 시스템 처리 |
|-------|-----------|
| 반려사유 공란 | 차단 (필수) |
| 반려사유 256자 초과 | 차단 |
| 정책 근거 | §2 |

### 3-7. [예외] 정산 문의 첨부 위반

| 케이스 | 시스템 처리 |
|-------|-----------|

### 3-8. [예외] 재확인필요 뱃지 노출

## 4. 컬리 임직원 시나리오 - CO(QA)

### 4-1. [정상] 정산 집계

| 항목 | 내용 |
|------|------|
| 권한 | CO(QA) 전용 |
| 사용자 행동 | 집계기준일·세무구분·공급사코드·마감일·처리자 입력 |
| 대상 조건 | 매입 대상 SKU **집계여부 = 'N'** + **정산포함 = '포함'** |
| 결과 | [집계완료] 상태 진입 |
| 정책 근거 | §5 |

### 4-2. [정상] 매입금액 산출

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 매입금액 산출 실행 |
| 제약 | **당월분만**, 공급사 개별 **최대 50개** |
| 계산식 | 매입금액 = 공급가액 + 부가세액 (상계금액 미차감) |
| 미처리 사유 | AP기간 다름 / 신규 정산코드 등 → 확인 후 재시도 |
| 정책 근거 | §7 |

### 4-3. [정상] 원천 상계금액 엑셀 등록

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 엑셀 등록 (최대 1,000건) |
| 시스템 처리 | 10자리 상계코드 자동 생성 |
| 제약 | 상계회차 1~3 / 상계유형 중복 불가 |
| 정책 근거 | §8-1 |

### 4-4. [정상] 구매론 관리

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 구매론 등록 |
| 시스템 처리 | 1 공급사 + 1 정산월 = 2+ 은행 가능 → **최종 하나의 한도금액으로 합산** |
| 차감 방식 | 은행 순서/금액 구분 없이 **총액 기준** |
| 정책 근거 | §8-2 |

### 4-5. [정상] 이월 상계금액 조정

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 입금요청금액 입력 |
| 입력값 제약 | **마이너스 1 이하 숫자 필수** |
| 중복 차단 | 동일 공급사 + 동일 정산월 중복 등록 불가 |
| 자동 마감 | 매월 1일 00시 |
| 정책 근거 | §8-3 |

### 4-6. [정상] 마감 후 변경 등록

| 유형 | 설명 | 공급사 노출 |
|------|------|-----------|
| 영업양수도 | 사업양수도에 따른 정산 조정 | ✅ |
| 수정지급 | 지급액 정정 | ✅ |
| 선지급 | 사전 지급 처리 | ✅ |
| 컬리조정 | 컬리 내부 조정 | ❌ **공급사 미노출** |

| 항목 | 제약 |
|------|------|
| 등록 대상 | **전월 마감 이후 건만** |
| 취소 가능 기간 | **전월 / 전전월만** (3개월 초과 불가) |
| 정책 근거 | §8-4, §10 |

### 4-7. [정상] 대금지급액 계산

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 회차별 최종지급금액 산출 |
| 제약 | 당월분만, 전체 또는 개별 공급사 (**최대 50개**) |
| 계산 흐름 | 매입금액 + 상계금액 + 구매론 → 회차별 최종지급금액 |
| 미변경 시 | 원천금액 변경 없으면 기존 대금지급액 유지 |
| 정책 근거 | §9 |

### 4-8. [정상] 정산 문의 답변

| 항목 | 내용 |
|------|------|
| 권한 | CO(QA) 전용 |
| 사용자 행동 | 공급사 문의에 답변 등록 |
| 제약 | **등록 후 수정 불가** |
| 시스템 처리 | 등록 즉시 공급사에 알림톡 자동 발송 |
| 정책 근거 | §11 |

### 4-9. [정상] 정산 취소

| 가능 상태 | 집계완료 / 승인요청 / 승인반려 / 승인완료(역발행실패·국세청실패·정발행대기) |
|---------|---------|
| 정책 근거 | §6 |

### 4-10. [정상] 계산서 발행 취소

| 항목 | 내용 |
|------|------|
| 가능 상태 | 승인완료 |
| 차단 조건 | **매입금액 산출 완료 시 불가** → 매입금액 먼저 취소 필요 |
| 정책 근거 | §6, §14 - ① |

### 4-11. [예외] 집계여부 'Y' 매입조정 시도

| 케이스 | 시스템 처리 |
|-------|-----------|
| 집계 완료된 SKU 금액 수정 | 차단 → **정산취소 후 수정** 필요 |
| 정책 근거 | §4 |

### 4-12. [예외] 매입금액 산출 후 정산취소·발행취소 시도

| 케이스 | 시스템 처리 |
|-------|-----------|
| 매입금액 산출 완료 후 정산취소 시도 | 차단 |
| 매입금액 산출 완료 후 계산서 발행취소 시도 | 차단 → **매입금액 취소 먼저** |
| 정책 근거 | §6, §14 - ① |

| 답변 등록 후 수정 시도 | 차단 |

## 11. 정산 처리 ASCII 흐름도

```
[CO부서: 매입조정] ── 정산포함 'Y' / 집계여부 'N' ──→ [CO(QA): 정산집계]
                                                              │
                                                              ▼
                                                       [집계완료]
                                                              │
                                                              ▼ 승인요청 발송
                                                       [승인요청]
                                                              │
                          ┌───────────────────────────────────┤
                          │                                   │
              공급사 [반려]                            공급사 [승인]
              (사유 256자 필수)                              │
                          ▼                                   ▼
                   [승인반려]                         [승인완료] + [발행대기]
                          │                                   │
                          │             ┌─────────────────────┤
                          │             ▼                     ▼
                          │      자동 역발행              정발행(공급사)
                          │             │                     │
                          │             ▼                     ▼
                          │      [계산서발행] ── 익일 ──→ [국세청전송]
                          │
                          ▼
              [정산취소 가능]: 집계완료/승인요청/승인반려
                              /승인완료(역발행실패·국세청실패·정발행대기)

[매입금액 산출(당월·≤50개)] ─→ [상계금액 등록] ─→ [구매론] ─→ [이월 상계]
          │                                                        │
          │                                          매월 1일 00시 자동 마감
          ▼
[마감 후 변경: 전월 마감 이후만 / 취소 ≤ 전전월]
          │
          ▼
[대금지급액 계산(당월·≤50개)] ─→ [대금지급현황]
                                        │
                                        ├─ 공급사 노출: 매입·상계·영업양수도·수정지급·선지급
                                        └─ 미노출: 구매론발행·마감변경·컬리조정

[공급사 정산문의] ─ CO(QA) 답변 등록(수정 불가) ─→ 알림톡 발송
```
