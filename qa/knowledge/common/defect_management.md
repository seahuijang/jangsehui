## 결함(Bug) 심각도 기준

| 심각도 | 기준 | 예시 |
|--------|------|------|
| **Critical** | 핵심 기능 사용 불가, 데이터 손실 발생 | 입고 처리 시 재고가 반영되지 않음 |
| **High** | 주요 기능 오작동, 정책 위반 처리 | 수량 초과인데 처리가 완료됨 |
| **Medium** | 기능은 동작하나 비정상적인 결과 | 완료 메시지가 오류 메시지로 표시됨 |
| **Low** | 사소한 표시 오류, UX 불편 | 버튼 위치가 요구사항과 다름 |

## 결함 보고서 작성 기준

### 좋은 결함 보고서 조건
1. **제목**: 결함 내용을 한 줄로 요약
2. **재현 단계**: 누구나 따라 했을 때 동일하게 재현 가능
3. **기대 결과 vs 실제 결과**: 명확한 비교
4. **증거**: 스크린샷 또는 영상 첨부

### Jira 버그 이슈 작성 형식
```
제목: [BUG] 현상 요약

재현 단계:
1. ~한다
2. ~한다
3. ~한다

기대 결과: ~해야 한다
실제 결과: ~가 발생한다

심각도: Critical / High / Medium / Low
첨부: 스크린샷
```

## 결함 처리 흐름

```
발견 → 등록(Jira) → 개발팀 확인 → 수정 → 재테스트 → 완료
                          ↓
                      재현불가 / 정책상 정상 → 종료
```

## 발주 담당자가 결함으로 분류해야 할 상황

- 요구사항 명세서에 명시된 정책과 다르게 동작하는 경우
- 수량·금액·상태 데이터가 처리 후 올바르게 반영되지 않는 경우
- 차단해야 할 상황에서 처리가 통과되는 경우
- 허용해야 할 상황에서 불필요하게 차단되는 경우
- 오류 메시지가 없거나 내용이 부정확한 경우
