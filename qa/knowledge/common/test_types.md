 유형

### 정상 케이스 (Normal Case)
- 비즈니스 정책을 준수하는 표준 흐름을 검증
- 예: 허용 수량 범위 내 입고 처리 → 정상 완료

### 예외 케이스 (Exception Case)
- 정책 위반, 오류 상황, 비정상 입력에 대한 시스템 반응 검증
- 예: 허용 수량 초과 입고 요청 → 차단 및 오류 메시지 표시

### 경계값 케이스 (Boundary Case)
- 허용 범위의 최솟값, 최댓값, 그 직전/직후 값 검증
- 예: 최대 입고 수량이 1000개일 때 → 999개(통과), 1000개(통과), 1001개(차단)

## 비기능 테스트 유형

### 권한 테스트
- 역할별 접근 가능/불가능 기능 검증
- 예: 일반 담당자는 발주 승인 불가, 팀장만 가능

### 중복 처리 테스트
- 동일 요청 반복 시 시스템 반응 검증
- 예: 같은 입고 번호로 중복 처리 시도 → 차단

### 상태 전이 테스트
- 허용된 상태 변경 흐름과 허용되지 않는 흐름 검증
- 예: 취소 완료된 발주 건 → 재처리 불가

## 테스트 기법

### 동등 분할 (Equivalence Partitioning)
유사한 결과를 내는 입력값을 그룹으로 묶어 대표값 1개만 테스트
```
입고 수량 정책: 1~100개(정상), 0이하(오류), 101이상(오류)
→ 대표 테스트값: 50개(정상), -1개(오류), 150개(오류)
```

### 경계값 분석 (Boundary Value Analysis)
경계 지점에서 오류가 가장 많이 발생하므로 경계값 집중 테스트
```
최대 허용 수량 100개 기준
→ 테스트값: 99, 100, 101
```

### 결정 테이블 (Decision Table)
여러 조건 조합에 따른 결과를 표로 정리
```
| 재고 있음 | 발주 승인 | 처리 결과 |
|---------|---------|---------|
| O       | O       | 입고 처리 |
| O       | X       | 대기     |
| X       | O       | 발주 요청 |
| X       | X       | 오류     |
```

### 상태 전이 (State Transition)
상태 변화 흐름을 기반으로 TC 도출
```
발주 생성 → 검토 중 → 승인 → 입고 완료
                   ↓
                  반려
```
